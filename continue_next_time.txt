HOW TO PICK THIS UP
-------------------
Whichever direction is chosen needs its own design -> decisions -> planning
cycle before implementation, same as every item in this thread so far: a
<topic>-design.md auditing what's actually in the code first (not assumed),
explicit decisions confirmed before coding, a <topic>-plan.md, then
TDD-first implementation with each decision logged in DECISIONS.md. Every
increment in this thread so far has turned up at least one real, previously
unknown finding during the audit step (a latent bug, a stale premise, a
design-vs-implementation gap) — the audit is not a formality.

Cross-referenced findings from this whole thread are also recorded in
agents/knowledge.db (project_id=1, name='harvey') as a chronological set of
'decision' observations dated 2026-07-12 and 2026-07-13 — useful for a quick
narrative recap without re-reading every design doc in full.
